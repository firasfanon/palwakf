# ربط الدليل بالمساعد الداخلي — PalWakf

## الهدف
هذا الملف يحدد كيف يستخدم المساعد الداخلي الدليل المؤسسي الحي دون تجاوز للصلاحيات.

## المبادئ
1. المساعد لا يعرض أي معلومة خارج النطاق الفعلي للمستخدم.
2. المساعد يجيب ضمن:
   - الدور التشغيلي
   - الأنظمة المسموحة
   - الوحدة أو الوحدات
   - الصلاحيات الفعلية
3. المساعد يستخدم هذا الدليل كمرجع وصفي، لا كمصدر قرار أمني مستقل.
4. القرار الأمني النهائي يأتي من:
   - RLS
   - RPC policy layer
   - AccessProfile
   - user scope assignments

## المدخلات اللازمة للمساعد
لكي يستخدم المساعد هذا الدليل بشكل صحيح، يجب تمرير:
- user_id
- username
- operational_role_key
- operational_role_label_ar
- unit_id / assigned_unit_ids
- assigned_system_keys
- direct permissions
- system roles
- scope assignments summary

## المخرجات المتوقعة
المساعد يستطيع:
- شرح ما يستطيع المستخدم فعله
- إرشاده إلى الصفحة المناسبة
- تفسير سبب ظهور/اختفاء إجراء ما
- اقتراح الخطوات التالية داخل نطاقه
- شرح علاقته بالنظام/الوحدة/الصلاحية

المساعد لا يستطيع:
- تجاوز الصلاحيات
- تنفيذ إجراء غير مسموح
- كشف مستخدمين أو وحدات أو بيانات خارج النطاق

## خرائط الاستخدام
### superuser
- مساعد حوكمي/إداري واسع

### power_admin
- مساعد متخصص في نظام واحد على مستوى الوحدات

### unit_admin
- مساعد لإدارة الوحدة وخدماتها ومستخدميها

### system_super_user
- مساعد لتشغيل نظام محدد داخل وحدة محددة

### employee
- مساعد تشغيلي يومي

### delegate_lawyer
- مساعد موجّه للنظام المفوض والوحدات الموكلة فقط

## توصية فنية
يجب أن يحمّل المساعد هذا الدليل من:
- docs/usage/
- config/usage_guide_manifest.json

ويستخدمه مع سياق المستخدم الحالي، لا كمعرفة عامة غير مقيدة.
